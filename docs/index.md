# nvimConfig — docs index

Typed catalog of the Neovim config's detailed docs. CLAUDE.md holds the lean
current-state; everything cold/reference lives here.

- [[architecture]] — initialization flow, plugin organization by category, key
  design patterns, per-feature implementation details, plus version requirements
  and external (non-Mason) dependencies.
- [[commands]] — every command + keybinding (testing, Lazy, Mason/LSP, DAP,
  neotest, Copilot, conform/lint, tree-sitter).
- [[workflows]] — how to add LSP servers / formatters / plugins / keybindings,
  edit-test-commit loop, common feature-adding patterns.
- [[troubleshooting]] — fixes per subsystem (plugins, LSP, formatting,
  tree-sitter, completion, DAP, neotest, Copilot, peek, diffview).
- [[changelog]] — version history / feature-addition log.
